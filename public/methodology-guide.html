2. CRISIS INTENSITY RISK — 위기강도 산출 방법

Level-Based Crisis Score
절대 가격 수준 기반 위기강도 지수
WTI 유가와 USD/KRW 환율의 절대 수준(Level)을 주요 신호로 사용합니다. 일별 변화율에만 의존하면 가격이 장기간 고공행진해도 신호가 0에 수렴하는 문제를 해결합니다.

기본 수준 신호 (Level Signals)

$$\text{oilLevelScore} = \min\!\Big(80,\;\max\!\big(0,\; \lfloor(WTI - 65) \times 1.5\rfloor\big)\Big)$$ $$\text{fxLevelScore} = \min\!\Big(60,\;\max\!\big(0,\; \lfloor(USDKRW - 1280) \times 0.22\rfloor\big)\Big)$$

WTI $65 이하 또는 환율 1,280원 이하이면 수준 신호 = 0. WTI $115이면 oilLevelScore = 75, 환율 1,450원이면 fxLevelScore = 37.

Method 1 — Gaussian / VAR 기반
⚙️ 모든 톱니바퀴가 맞물려 돌아간다
VAR(벡터자기회귀) 모델에 정규분포(Gaussian) 가정을 얹은 방식. 모든 변수가 서로 영향을 주고받는다고 가정하여 연립방정식으로 동시 모델링합니다.
$$\Delta CC_h = \sum_{k} \text{shock}_k \times w_k \times \exp\!\left(-\frac{(h - \mu_k)^2}{2\sigma_k^2}\right) \times 0.1$$

$w_k$: 드라이버 가중치, $\mu_k$: peak lag(최대 반응 시점), $\sigma_k$: 확산폭.
관성 감쇠: $\max(0.1,\; \overline{\Delta CC} \cdot e^{-h/10})$

장점

  • 변수 간 상호작용 포착 (환율↑→수입물가↑→건설비↑ 연쇄)
  • 데이터 없이도 작동 — 파라미터만 설정하면 즉시 결과
  • 소표본에서도 안정적, 계산 즉시 완료

한계

  • 구조적 충격 식별 어려움 (식별 문제)
  • 가중치가 추정이 아닌 가정 — 비선형 관계 놓침
  • 구조 변화(전쟁, 정책 전환)에 적응 불가
권장 사용 시점: 데이터가 부족하거나 빠른 방향성 판단이 필요할 때. Baseline 기준선 역할.
Method 2 — ARDL-OLS
🌤 오늘 날씨 = 어제 날씨 + 어제 기압 + 그제 기압
AutoRegressive Distributed Lag — 변수의 과거(lag)와 다른 변수의 과거를 함께 고려합니다. 단기·장기 효과를 분리하고 공적분(cointegration) 관계 분석에 적합합니다.
$$\Delta CC_t = \alpha + \sum_{k=1}^{K} \sum_{l=0}^{L} \beta_{k,l} \cdot \Delta X_{k,t-l} + \varepsilon_t$$

$K=8$ 드라이버, $L=4$ 최대 시차 → 총 41개 계수 OLS 추정. 단일 방정식 중심이므로 피드백 효과는 제한적입니다.
IRF 계산 시 maxLag 초과 구간은 지수 감쇠로 연결:

$$\text{IRF}(h) = \begin{cases} \beta_{k,h} & h \leq L_{\max} \\ \beta_{k,L_{\max}} \cdot e^{-0.4(h - L_{\max})} & h > L_{\max} \end{cases}$$

장점

  • 단기·장기 효과 분리 가능
  • 공적분 관계 분석에 적합
  • R²로 모델 적합도 정량 검증
  • 분석 기간 변경 시 계수 자동 재추정

한계

  • 소표본(n < 41)에서 과적합 위험
  • 재귀 전망 시 오차 누적
  • 단일 방정식 → 피드백 효과 제한적
IRF 절벽(Cliff) 수정: h > maxLag에서 0으로 끊지 않고 지수 감쇠 처리 → 전망 경로의 급격한 단절 방지.
권장 사용 시점: 전체 5년 기간(n=60)에서 장기 평균 구조 파악. n < 40이면 Gaussian 폴백 권장.
Method 3 — Local Projections (Jordà, 2005)
📷 1년 후, 2년 후, 3년 후를 각각 별도 카메라로 찍는다
Jordà(2005) 제안, 최근 가장 많이 쓰이는 방법론. h시점 이후 반응을 h번 각각 별도 회귀분석으로 직접 추정합니다. VAR/ARDL의 재귀 누적 오차를 원천 차단합니다.
$$y_{t+h} - y_t = \alpha_h + \sum_{k=1}^{K} \beta_{h,k} \cdot \Delta X_{k,t} + \varepsilon_{t+h} \quad \forall\, h = 1,\ldots,12$$

12개의 독립 OLS 회귀. $\beta_{h,k}$: "드라이버 $k$가 1단위 변화할 때 $h$개월 후 누적 영향".
모델 오류(misspecification)에 강건. 비선형·비대칭 효과 포착 용이.

장점

  • 오차 비누적 — 장기 전망에서도 신뢰구간 안정
  • 비선형·비대칭 충격 반응 포착
  • 모델 misspecification에 강건
  • 직관적 해석: "$\beta_{6,\text{oil}}$은 유가 1% 충격의 6개월 후 누적 영향"

한계

  • 단기(1~3M) 추정치 분산이 ARDL보다 클 수 있음
  • h↑ 시 유효 표본 감소 → 정밀도 하락
  • 소표본(< 20)에서 불안정
권장 사용 시점: 전쟁·금융위기 등 비선형 대형 충격 분석. 최근 학계 주류 방법론. 방향성 중심 해석 권장.

6. PF 금리 영향 산출 방법

Project Finance Rate Impact
신용등급별 PF 금리 상승폭 추정
기준금리 충격과 건설공사비 상승에 따른 리스크 프리미엄을 단일 공식으로 통합합니다.

수식

$$\Delta r_{\text{PF}} = \Delta r_{\text{policy}} + \Delta CC_{12} \times \text{baseSpread} \times \alpha$$

여기서 $\Delta r_{\text{policy}}$는 기준금리 충격(단위: %p), $\Delta CC_{12}$는 12개월 건설비 변화율, $\text{baseSpread}$는 등급별 기초 스프레드입니다.

파라미터상승 충격하락 충격의미

모델 캐싱

9. 분석 결과 해석 가이드

PRESSURE BADGE 해석
HIGH PRESSURE — 복합 점수 > 3.0. 건설비 상승 압력이 강하게 작용 중.
WATCH — 1.5~3.0. 일부 변수에서 압력 신호 감지.
STABLE — < 1.5. 유의미한 상승 압력 없음.
3M/6M/12M 추가압력 해석
+2.5% (3M) → 향후 3개월간 건설공사비 지수가 현재 대비 2.5% 추가 상승 예상
-0.3% (6M) → 6개월 시점에서 충격이 흡수되어 소폭 하락 전환 예상
3방법 결과가 동일 방향 → 확신↑, 서로 다르면 불확실성 구간
ATTRIBUTION TABLE 해석
기여도(%) — 해당 변수가 전체 공사비 변동의 몇 %를 설명하는지.
반영시차 — 충격 후 최대 영향까지 걸리는 개월 수 (peakLag).
전달경로 — 충격이 건설공사비까지 도달하는 중간 경로.
기여도가 가장 큰 변수 = 핵심 관리 대상. 시차가 긴 변수는 선제 대응 가능.
상관관계 vs 인과관계 주의
상관계수(r)가 높다고 인과관계를 의미하지 않습니다.
r > 0.7 — 강한 양의 상관. 함께 움직이는 경향 강함.
r < -0.7 — 강한 음의 상관. 역방향으로 함께 움직임.
인과 방향 확인에는 Granger 인과검정 또는 구조 VAR이 필요합니다.

10. System Dynamics와의 관계

SD는 "구조"를 그리고, 계량경제는 "크기"를 잰다.
두 접근법은 경쟁 관계가 아니라 보완 관계입니다.

계량경제 3종 (Gaussian · ARDL · LP)System Dynamics
목적과거 데이터로 관계 추정인과 구조로 미래 시뮬레이션
출발점데이터이론 / 구조
인과관계통계적 추론 (암묵적)명시적 설계 (루프로 그림)
피드백 루프암묵적 (선형 근사)핵심 구조 (강화·균형 루프)
비선형LP가 부분적으로 허용완전 허용 (설계 그대로)
비유사진 분석가 (데이터를 읽는다)설계 엔지니어 (구조를 그린다)

충격 반응 관점에서 비교
LP — 데이터에서 충격반응 추출 (측정)
VAR — 변수 간 상호작용 포착 (선형 피드백)
ARDL — 장단기 관계 추정 (단순 피드백)
SD — 왜 그런 반응 모양인지 피드백 루프로 설명
위기 모니터링에서의 통합
① SD 모델로 위기 확산 루프 설계
② LP/ARDL로 실제 확산 속도·크기 추정
③ 추정된 파라미터를 SD 모델에 주입
④ SD로 정책 피드백 포함 시나리오 시뮬레이션
→ 대시보드의 System Dynamics 탭이 이 역할
한 줄 정리: SD로 메커니즘을 설계하고 → LP/ARDL로 파라미터를 추정하고 → SD로 시뮬레이션한다.
계량경제가 "얼마나"를 측정하면, System Dynamics는 "왜"를 설명합니다.
